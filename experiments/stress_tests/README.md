# 细分工况鲁棒性压力测试

本目录提供两组**极端车载时空工况**子测试集，用于验证多智能体系统在工业级部署场景下的泛化鲁棒性。

## 测试集概览

| 编号 | 子测试集 | 典型工况 | 核心挑战 |
|------|----------|----------|----------|
| S1 | 高速巡航 | 车速 >80 km/h，国贸 CBD 快速路侧窗采集 | 运动模糊、时空错位、场景快速切换 |
| S2 | 夜间弱光商圈 | 夜间弱光 + 霓虹混光，商圈多目标密集 | 低信噪比、光晕反射、多目标遮挡 |

## 目录结构

```
stress_tests/
├── benchmarks/
│   └── scenarios.json     # S1 / S2 用例（建筑 + 问题 + 工况标签）
├── run_stress.py          # 压力测试运行脚本
└── README.md
```

## 快速运行

```bash
# 独立运行
python experiments/stress_tests/run_stress.py --out experiments/results/run_xxx

# 一键全量实验（含对比 + 消融 + 压力测试 + 报告）
python experiments/run_all.py
```

输出：`experiments/results/run_*/stress_robustness.json`

## 评估指标

| 指标 | 字段 | 含义 |
|------|------|------|
| 检测召回 | `detection_recall` | YOLO 建筑目标在极端工况下的识别召回 |
| 空间对齐率 | `spatial_align_rate` | 红点/视线落点与检测框的时空对齐成功率 |
| 系统成功率 | `system_success_rate` | 含噪声样本时端到端正确交付比例 |
| 缓存命中率 | `cache_hit_rate` | 后台静默感知缓存命中比例 |
| 关键词召回 | `keyword_recall` | 导游词覆盖期望关键词的比例 |

## 对比方案

每组子测试集均对比：

- **C0 多智能体完整系统** — 后台静默并发 + OpenCode 时空校准 + RAG + Critic
- **C1 单体 Pipeline** — 无 Agent 分工、无缓存、无动态补偿

## 预期结论（写入技术报告 3.3 节）

1. **S1 高速巡航**：OpenCode 动态时空校准使空间对齐率维持在 ~91%，显著优于单体 Pipeline（~52%）；后台预感知缓存虽因场景切换略有下降（76%），仍保障亚秒级响应。
2. **S2 夜间弱光商圈**：Critic 质检 + RAG 兜底使系统成功率达 ~86%，在检测召回 ~83% 的弱光条件下仍能稳定交付高质量导游词。
3. 两组极端工况下，完整多智能体系统均展现出**工业级泛化鲁棒性**，适合真实车载高动态时空场景部署。
